01 —

Credibility Crimes

FATAL
Testimonials
The First Reviewer is the Owner
The very first testimonial — introduced as "Реальные проекты, реальные люди" (Real projects, real people) — is from Павел Пронин, owner of a country house in Novaya Moskva. The GitHub repo is proninpaul-afk. Paul Pronin. AFK. Writing his own five-star review. The audacity is genuinely impressive.
/* "Real projects. Real people." */ <strong>Павел Пронин</strong> <span>Владелец загородного дома, Новая Москва</span> // github.com/proninpaul-afk/Rock
FATAL
Cinema Claims
"We work with Mosfilm" — Zero Photos
An entire section dedicated to cinematic prestige — Mosfilm, Moskino, Kinopoisk, Okko — with four dedicated film cards. Every single one has a placeholder that says "фото со съёмок" (photo from set). You're telling me you've worked on Russian blockbusters shot in 4K, and you cannot produce one photograph from a set? Not one?
<div class="film-card-photo-placeholder"> фото со съёмок ← × 4 </div> // "Our rocks survived 4K scrutiny." // "Our photos did not survive any scrutiny at all."
FATAL
Math
15 Years. Since 2004. Pick One.
The trust strip proudly announces "15 лет в оформлении интерьеров" (15 years in interior decoration). The footer says "с 2004 года" (since 2004). It's 2026. That's 22 years. Either they forgot to update the trust strip, or time works differently in the rock business. One of these is lying and they're right next to each other on the page.
trust-strip: "15 лет" в оформлении интерьеров footer: "с 2004 года" actual math: 2026 − 2004 = 22 года // 7 years of experience unaccounted for.
MEDIUM
Instagram
"Инстаграмный фон" — in Russia
The restaurant/lobby card promises an "инстаграмный фон" (Instagram background) that generates free marketing. Instagram has been banned in Russia since March 2022 as an "extremist organization." Selling premium luxury rock work, but the copywriting hasn't been updated in 4 years.

"Our rocks singlehandedly survived 4K camera scrutiny. Our contact form does not survive a single click."

— Rock&Stone, unknowingly
02 —

Technical Disasters

FATAL
Contact Form
The Form is a Prop
There is an elaborate contact form with validation, loading states, error handling, a success screen, and even an animation spinner. It sends to a Telegram bot. The bot token is an empty string. The form never sends anything — it silently falls back to opening a Telegram link. The form is decorative. Like a movie prop. Fitting, perhaps.
const TELEGRAM_BOT_TOKEN = ''; // "например: '1234567890:AAH_xxx...'" const TELEGRAM_CHAT_ID = ''; // "например: '123456789'" // ~80 lines of form logic // → always falls back to opening t.me link // → the spinner spins for no one
FATAL
Architecture
2,503 Lines. One File. No Regrets.
The entire website — CSS, HTML, JS, inline SVGs, base64 encoded noise textures, a complete Telegram API integration, FAQ accordion, carousel with drag support, lightbox, tab switcher, scroll observer, sticky CTA logic — all lovingly packed into a single 2,503-line index.html. An absolute unit of a file. Scrolling the source code is itself a journey.
$ wc -l index.html 2503 index.html // CSS: ~900 lines // HTML: ~500 lines // JavaScript: ~300 lines of working code // Form logic: ~200 lines sending to nowhere

Branding
The GitHub Username is proninpaul-afk
You are selling premium handcrafted decorative rock installations worth 200,000–350,000 rubles. You promise to respond in "15 minutes". Your GitHub username ends in afk. Away. From. Keyboard. This is not a roast observation — this is a lifestyle contradiction.
github.com/proninpaul-afk/Rock // site promise: "ответим за 15 минут" // username reality: -afk
MEDIUM
CSS Variables
--black: #0f0f0f — Not Actually Black
A purely symbolic complaint, but: the CSS variable named --black is #0f0f0f. True black is #000000. This is the lightest shade of grey they could've gotten away with calling black. The site is built on a lie, structurally, from line one.
--black: #0f0f0f; /* coward grey */ --stone: #1a1a1a; --stone-light: #252525; /* none of these are black. rock sold separately. */
MILD
Design Detail
A Noise Texture at 3% Opacity
There is a base64-encoded SVG fractal noise texture applied across multiple sections via a .noise::before pseudo-element. Its opacity is 0.03. It is completely invisible to 99% of users on 99% of screens. But it's there. Someone cared. We respect the commitment even as we question the impact.
.noise::before { background-image: url("data:image/svg+xml, ... feTurbulence ..."); opacity: 0.03; /* ghost texture, whisper aesthetic */ } // human eye threshold: ~0.05 // this texture: technically present
MILD
Contact Options
There is a max.ru Link
Alongside Telegram and Phone, the contact section features a link to max.ru — a messenger approximately 11 people use. The URL is not a clean handle. It is a 70-character authentication token that looks like a cryptographic hash. It is presented in a premium gold-bordered card next to a Playfair Display headline.

03 —

What's Actually Good

Real Photography

The portfolio images are genuinely impressive. The work speaks for itself — textured, tactile, cinematic. The product is real and the photos prove it. Put more of these everywhere and delete half the copy.

Solid Visual System

Dark background, warm cream, gold accents — the color palette is coherent and appropriate for a premium craft product. Playfair Display + Inter is a reliable pairing. The bones are good.

Thoughtful Interactions

The carousel has drag support. The FAQ accordion animates properly. The sticky CTA logic is nuanced (hides near contact section). Someone actually thought about UX. It shows.

Smart Section Structure

Problem → Solution → Origin → Process → Segments → Portfolio → Testimonials → FAQ → Pricing → CTA is a well-structured conversion funnel. The narrative logic is sound, even if the content sometimes isn't.

The Cinema Angle is Genuinely Great

"We make rocks for movies, now we make them for your home" is a fantastic USP. It's differentiated, credible in principle, and memorable. Just needs real photos to back it up.

Responsive Layouts

Media queries are applied consistently. The mobile grid fallbacks are sensible. This is a single-developer project and the responsive work is better than many agency sites three times the budget.

Final Verdict

Good bones, shaky foundations — fix the trust layer and this site works

The design direction is solid. The product is clearly real and good. But the credibility infrastructure — fake-ish testimonials, empty film placeholders, a broken contact form, and contradictory dates — undermines every legitimate claim. The rocks survive 4K. The trust signals don't survive inspection.

5.2 / 10
Would commission a rock. Would not trust the reviews.
